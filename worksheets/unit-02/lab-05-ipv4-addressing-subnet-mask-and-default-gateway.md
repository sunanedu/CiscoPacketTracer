## 1. เรื่องของใบงาน

ใบงานนี้เป็นการศึกษาองค์ประกอบสำคัญของการกำหนดค่า IPv4 ได้แก่ **IP Address**, **Subnet Mask** และ **Default Gateway** ผู้เรียนจะสร้างเครือข่าย LAN สองเครือข่าย เชื่อมต่อกันด้วย Router แล้วทดสอบการสื่อสารภายใน LAN เดียวกันและข้ามไปยังอีก LAN

Router ในแล็บนี้จะกำหนดค่าผ่านแท็บ **Config** ของ Packet Tracer เพื่อให้ผู้เรียนมุ่งทำความเข้าใจเรื่องหมายเลขเครือข่ายก่อน การกำหนด Router ผ่าน Cisco IOS CLI จะเรียนในแล็บภายหลัง

ผู้เรียนจะทดลองลบ Default Gateway และเปลี่ยน Subnet Mask ให้ผิดชั่วคราว เพื่อสังเกตผลที่เกิดขึ้นและฝึกวิเคราะห์สาเหตุอย่างเป็นลำดับ

## 2. เป้าหมายการเรียนรู้

เมื่อทำใบงานเสร็จ ผู้เรียนควรสามารถทำสิ่งต่อไปนี้ได้

1. อธิบายหน้าที่ของ IPv4 Address, Subnet Mask และ Default Gateway ได้
2. แยกส่วน Network และ Host ของเครือข่าย `/24` ได้
3. ระบุ Network Address, ช่วง Host และ Broadcast Address ของ `/24` ได้
4. ตรวจสอบว่า IPv4 Address สองหมายเลขอยู่เครือข่ายเดียวกันหรือไม่เมื่อใช้ `/24` ได้
5. กำหนด IPv4 แบบ Static ให้ PC ได้
6. กำหนด IP Address และเปิด Interface ของ Router ผ่านแท็บ Config ได้
7. อธิบายว่าอุปกรณ์ใน LAN เดียวกันสื่อสารกันได้โดยไม่ใช้ Default Gateway ได้
8. อธิบายว่าอุปกรณ์ต่างเครือข่ายต้องส่งข้อมูลผ่าน Default Gateway ได้
9. ตรวจสอบค่าด้วย `ipconfig` และทดสอบด้วย `ping` ได้
10. แก้ไขปัญหาที่เกิดจาก Default Gateway หรือ Subnet Mask ไม่ถูกต้องได้

## 3. ความรู้พื้นฐานก่อนเริ่ม

- ตรวจสอบสถานะลิงก์และแก้ไขการต่อสายเบื้องต้นได้

## 4. แนวคิดพื้นฐาน

### 4.1 IPv4 Address

IPv4 Address มีขนาด 32 บิต แสดงเป็นตัวเลขฐานสิบ 4 ชุด แต่ละชุดมีค่าตั้งแต่ 0 ถึง 255 และคั่นด้วยจุด เช่น

```text
192.168.10.10
```

IPv4 Address ของอุปกรณ์ในเครือข่ายเดียวกันต้องไม่ซ้ำกัน

### 4.2 Subnet Mask และ Prefix Length

Subnet Mask บอกว่าส่วนใดของ IPv4 Address เป็นส่วน Network และส่วนใดเป็นส่วน Host

ในแล็บนี้ใช้

```text
Subnet Mask:   255.255.255.0
Prefix Length: /24
```

`/24` หมายถึง 24 บิตแรกเป็นส่วน Network และ 8 บิตสุดท้ายเป็นส่วน Host

ตัวอย่าง

```text
IP Address:      192.168.10.10
Subnet Mask:     255.255.255.0
Network Portion: 192.168.10
Host Portion:                10
Network Address: 192.168.10.0
```

### 4.3 Network Address

Network Address ใช้ระบุเครือข่าย ไม่สามารถกำหนดให้กับ PC หรือ Interface ของ Router ได้

ตัวอย่าง

```text
192.168.10.0/24
```

### 4.4 Broadcast Address

Broadcast Address ใช้ส่งข้อมูลถึงทุก Host ในเครือข่ายย่อยนั้น ไม่สามารถกำหนดให้กับอุปกรณ์ทั่วไปได้

ตัวอย่าง

```text
192.168.10.255
```

### 4.5 Default Gateway

Default Gateway คือ IP Address ของ Router Interface ที่อยู่ใน LAN เดียวกับเครื่องต้นทาง เมื่อเครื่องต้องส่งข้อมูลไปยัง Network อื่น จะส่ง Frame ไปยัง MAC Address ของ Default Gateway

หลักสำคัญมีดังนี้

- Default Gateway ต้องอยู่ Subnet เดียวกับ Host
- Host ที่ติดต่อกันภายใน Subnet เดียวกันไม่จำเป็นต้องผ่าน Default Gateway
- Host ที่ต้องการติดต่อ Network อื่นต้องมี Default Gateway ที่ถูกต้อง
- การกรอกหมายเลข Gateway อย่างเดียวไม่เพียงพอ ต้องมี Router Interface ที่ใช้หมายเลขนั้นและทำงานอยู่จริง

### 4.6 Private IPv4 Address

เครือข่ายที่ใช้ในแล็บคือ `192.168.10.0/24` และ `192.168.20.0/24` ซึ่งอยู่ในช่วง Private IPv4 Address `192.168.0.0/16` เหมาะสำหรับใช้งานภายในองค์กร บ้าน หรือห้องปฏิบัติการ และไม่ถูกส่งต่อโดยตรงบนอินเทอร์เน็ตสาธารณะ

### 4.7 ตารางเปรียบเทียบสองเครือข่าย

| รายการ | LAN A | LAN B |
|---|---|---|
| Network Address | `192.168.10.0` | `192.168.20.0` |
| Prefix Length | `/24` | `/24` |
| Subnet Mask | `255.255.255.0` | `255.255.255.0` |
| Host ที่ใช้งานได้ | `.1` ถึง `.254` | `.1` ถึง `.254` |
| Broadcast Address | `192.168.10.255` | `192.168.20.255` |
| Default Gateway | `192.168.10.1` | `192.168.20.1` |

### 5.1 ซอฟต์แวร์

### 5.2 อุปกรณ์จำลองใน Packet Tracer

| ลำดับ | ประเภทอุปกรณ์ | รุ่นที่แนะนำ | จำนวน | ชื่อที่กำหนด |
|---:|---|---|---:|---|

| 3 | PC | PC-PT | 3 | `PC1`, `PC2`, `PC3` |

## 6. แผนผังเครือข่าย

```text
                  LAN A: 192.168.10.0/24

 PC1 Fa0 -------- Fa0/1 SW1 Gi0/1 -------- Gi0/0 R1
 .10                                      .1    |
                                                 | Gi0/1
 PC2 Fa0 -------- Fa0/2 SW1                      | .1
 .20                                             |
                                                 |        LAN B: 192.168.20.0/24
                                                 |
                                      Gi0/1 SW2 Fa0/1 -------- Fa0 PC3
                                                                    .10
```

มุมมองอย่างย่อ

```text
PC1 ----\
         SW1 ---- R1 ---- SW2 ---- PC3
PC2 ----/

LAN A          Router          LAN B
```

สายทุกเส้นในแล็บนี้เป็น **Copper Straight-Through**

## 15. แนวทางแก้ไขปัญหาเบื้องต้น

ตรวจสอบตามลำดับจากใกล้ตัวไปไกลตัว

```text
1. ตรวจ IP และ Mask ของเครื่องตนเอง
2. Ping IP ของตนเอง
3. Ping Host ใน LAN เดียวกัน
4. Ping Default Gateway
5. Ping Host ใน LAN ปลายทาง
6. ตรวจค่าของ Host ปลายทางและ Gateway ขากลับ
```

| อาการ | สาเหตุที่เป็นไปได้ | วิธีตรวจสอบหรือแก้ไข |
|---|---|---|
| พอร์ต Router เป็นสีแดง | Port Status ยัง Off หรือสาย/พอร์ตผิด | เปิด Config ของ Interface และตั้ง Port Status เป็น On |
| PC1 Ping PC2 ไม่ได้ | IP/Mask สาย หรือพอร์ตของ LAN A ผิด | ตรวจ `ipconfig` และลิงก์ PC–SW1 |
| PC1 Ping PC2 ได้แต่ Ping Gateway ไม่ได้ | R1 G0/0 ไม่ทำงานหรือกำหนด IP ผิด | ตรวจ `192.168.10.1/24` และ Port Status |
| PC1 Ping Gateway ได้แต่ Ping PC3 ไม่ได้ | PC1/PC3 Gateway ผิด, R1 G0/1 ผิด หรือ LAN B มีปัญหา | ตรวจค่าทั้งสองฝั่งตามตาราง |
| PC1 ไม่มี Gateway แล้ว Ping PC2 ได้ | เป็นผลที่ถูกต้อง เพราะ PC2 อยู่ LAN เดียวกัน | ไม่ต้องแก้จนกว่าจะทดสอบต่าง LAN |
| PC1 ไม่มี Gateway แล้ว Ping PC3 ไม่ได้ | ไม่มี Next Hop สำหรับ Network อื่น | คืน Gateway เป็น `192.168.10.1` |
| PC1 ใช้ `/16` แล้ว Ping PC3 ไม่ได้ | PC1 เข้าใจผิดว่า PC3 เป็น Local Host | คืน Mask เป็น `255.255.255.0` |
| Ping ไปได้แต่ Ping กลับไม่ได้ | IP, Mask หรือ Gateway ของฝั่งปลายทางผิด | ตรวจค่า PC ปลายทางและเส้นทางขากลับ |
| Ping ครั้งแรกสูญหายหนึ่ง Packet | อุปกรณ์กำลังทำ ARP | Ping ซ้ำแล้วเปรียบเทียบผล |

| IPv4 Address | Network Address | ประเภท | กำหนดให้อุปกรณ์ได้หรือไม่ |
|---|---|---|---|
| `192.168.10.0` | `192.168.10.0` | Network | ไม่ได้ |
| `192.168.10.1` | `192.168.10.0` | Host | ได้ |
| `192.168.10.10` | `192.168.10.0` | Host | ได้ |
| `192.168.10.254` | `192.168.10.0` | Host | ได้ |

| `192.168.20.10` | `192.168.20.0` | Host | ได้ |

คู่ที่ 1 และ 3 อยู่ Network เดียวกัน ส่วนคู่ที่ 2 และ 4 อยู่คนละ Network เมื่อใช้ `/24`

### แนวคำตอบคำถามท้ายแล็บ

1. IP Address ระบุหมายเลขของ Interface ส่วน Subnet Mask ใช้แยกส่วน Network และ Host
2. 24 บิตแรกเป็น Network Portion และตรงกับ `255.255.255.0`
3. Network คือ `192.168.20.0` และ Broadcast คือ `192.168.20.255`
4. `.0` เป็น Network Address และ `.255` เป็น Broadcast Address ของ `/24` จึงสงวนไว้
5. ต้องอยู่ Subnet เดียวกับ Host เพื่อให้ Host ติดต่อ Gateway โดยตรงผ่าน Layer 2 ได้
6. PC2 อยู่ Network เดียวกับ PC1 จึงส่ง Frame ถึงกันผ่าน Switch โดยตรง
7. PC3 อยู่ต่าง Network และ PC1 ไม่มี Router หรือ Next Hop ที่จะรับ Packet ไปส่งต่อ
8. ไม่ต้องมี Static Route เพิ่ม เพราะทั้งสอง Network เชื่อมต่อโดยตรงกับ Interface ของ R1
9. `/16` ใช้ `192.168` เป็น Network Portion จึงจัดทั้ง `192.168.10.10` และ `192.168.20.10` เป็น Network เดียวกัน
10. ตรวจ Interface ฝั่ง LAN B, IP/Mask/Gateway ของ PC3 และเส้นทางขากลับ

### ข้อสังเกตสำหรับผู้สอน

- จุดสำคัญของแล็บคือให้ผู้เรียนอธิบายได้ว่า Same-LAN ไม่ใช้ Gateway แต่ Inter-LAN ต้องใช้ Gateway
- การกำหนด Router ผ่าน Config เป็นเพียงสะพานไปสู่ IOS CLI ไม่ควรลงรายละเอียดคำสั่ง Router ในแล็บนี้
- ก่อนรับไฟล์ ตรวจว่า PC1 คืน Mask เป็น `/24` และ Gateway เป็น `192.168.10.1` แล้ว
- Router มี Route ของทั้งสอง Network แบบ Directly Connected จึงส่งต่อระหว่างสอง LAN ได้โดยไม่ต้องกำหนด Static Route
