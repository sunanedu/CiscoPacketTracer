## 1. เรื่องของใบงาน

ใบงานนี้เป็นการแบ่งเครือข่าย `192.168.100.0/24` ให้เหมาะกับจำนวนอุปกรณ์ของแต่ละแผนกด้วย **Subnetting** และ **Variable Length Subnet Mask หรือ VLSM**

ผู้เรียนจะคำนวณขนาด Subnet จากจำนวน Host เรียงความต้องการจากมากไปน้อย หา Network Address, ช่วง Host, Broadcast Address และ Subnet Mask จากนั้นนำผลลัพธ์ไปกำหนดให้ Router และ PC ใน Cisco Packet Tracer

Router ในแล็บนี้กำหนดค่าผ่านแท็บ **Config** เพื่อให้ผู้เรียนมุ่งเน้นการคำนวณ Subnet การตั้งค่าด้วย Cisco IOS CLI จะเริ่มในแล็บถัดไป

## 2. สถานการณ์จำลอง

องค์กรหนึ่งได้รับเครือข่าย

```text
192.168.100.0/24
```

และต้องการแบ่งให้หน่วยงานดังต่อไปนี้

| ลำดับความต้องการ | หน่วยงาน | จำนวน Host ที่ต้องรองรับ |
|---:|---|---:|
| 1 | ADMIN | 50 |
| 2 | SALES | 25 |
| 3 | IT | 10 |
| 4 | เครือข่าย Point-to-Point ในอนาคต | 2 |

แต่ละหน่วยงานควรได้รับ Subnet ที่เล็กที่สุดซึ่งยังรองรับจำนวน Host ได้ และห้ามมีช่วงหมายเลขซ้อนทับกัน

## 3. เป้าหมายการเรียนรู้

เมื่อทำใบงานเสร็จ ผู้เรียนควรสามารถทำสิ่งต่อไปนี้ได้

1. อธิบายเหตุผลของการแบ่งเครือข่ายเป็น Subnet ได้
2. อธิบายความแตกต่างระหว่าง Fixed Length Subnet Mask และ VLSM ได้
3. คำนวณจำนวน Host ที่ใช้งานได้จากจำนวน Host Bit ได้
4. เลือก Prefix Length ที่เล็กที่สุดซึ่งรองรับจำนวน Host ตามโจทย์ได้
5. แปลง Prefix `/26`, `/27`, `/28` และ `/30` เป็น Subnet Mask ได้
6. คำนวณ Block Size, Network Address และ Broadcast Address ได้
7. จัดสรร VLSM จากความต้องการมากไปน้อยโดยไม่ให้ Subnet ซ้อนทับกันได้
8. กำหนด IP Address, Subnet Mask และ Default Gateway จากแผน VLSM ได้
9. ทดสอบการเชื่อมต่อระหว่าง Subnet ที่เชื่อมตรงกับ Router ได้
10. ตรวจพบและแก้ไข Subnet Mask ที่ไม่ตรงกับแผนได้

## 4. ความรู้พื้นฐานก่อนเริ่ม

ผู้เรียนควรสามารถ

- ระบุ Network Address, Host Address และ Broadcast Address ของ `/24` ได้
- อธิบายหน้าที่ของ IP Address, Subnet Mask และ Default Gateway ได้
- กำหนด Router Interface ผ่านแท็บ Config ได้
- ใช้ `ipconfig` และ `ping` ได้
- ใช้เครื่องคิดเลขเลขยกกำลังพื้นฐานได้

## 5. แนวคิดพื้นฐาน

### 5.1 ทำไมต้องแบ่ง Subnet

การแบ่ง Subnet ช่วย

- แยก Broadcast Domain
- จัดกลุ่มอุปกรณ์ตามหน่วยงานหรือหน้าที่
- ลด Traffic แบบ Broadcast ในแต่ละเครือข่าย
- ใช้หมายเลข IP อย่างเป็นระบบ
- กำหนดนโยบาย Routing และความปลอดภัยได้ง่ายขึ้น

### 5.2 FLSM และ VLSM

| วิธี | ความหมาย | ลักษณะ |
|---|---|---|
| FLSM | Fixed Length Subnet Mask | ทุก Subnet ใช้ Mask ขนาดเดียวกัน |
| VLSM | Variable Length Subnet Mask | แต่ละ Subnet ใช้ Mask ต่างกันตามจำนวน Host |

หากทุกแผนกได้รับ `/26` เท่ากัน แผนก IT ที่ต้องการเพียง 10 Host จะเหลือหมายเลขจำนวนมาก VLSM จึงเลือก `/28` ที่มี 14 Usable Hosts ให้ IT แทน

### 5.3 สูตรจำนวน Host

```text
จำนวน Address ทั้งหมด = 2^h
จำนวน Host ที่ใช้งานได้ = 2^h - 2
```

โดย `h` คือจำนวน Host Bit ที่เหลืออยู่

ลบ 2 เพราะสงวน

- 1 Address สำหรับ Network Address
- 1 Address สำหรับ Broadcast Address

### 5.4 การหา Prefix Length

```text
Prefix Length = 32 - จำนวน Host Bit
```

ตัวอย่าง ต้องการ 50 Hosts

```text
2^5 - 2 = 30    ไม่เพียงพอ
2^6 - 2 = 62    เพียงพอ
Prefix = 32 - 6 = /26
```

### 5.5 ตาราง Prefix ที่ใช้ในแล็บ

| Prefix | Subnet Mask | Host Bits | Address ทั้งหมด | Usable Hosts | Block Size ใน Octet สุดท้าย |
|---|---|---:|---:|---:|---:|
| `/24` | `255.255.255.0` | 8 | 256 | 254 | 256 |
| `/25` | `255.255.255.128` | 7 | 128 | 126 | 128 |
| `/26` | `255.255.255.192` | 6 | 64 | 62 | 64 |
| `/27` | `255.255.255.224` | 5 | 32 | 30 | 32 |
| `/28` | `255.255.255.240` | 4 | 16 | 14 | 16 |
| `/29` | `255.255.255.248` | 3 | 8 | 6 | 8 |
| `/30` | `255.255.255.252` | 2 | 4 | 2 | 4 |

### 5.6 การหา Block Size

เมื่อ Prefix อยู่ใน Octet สุดท้าย สามารถหา Block Size ได้จาก

```text
Block Size = 256 - ค่า Subnet Mask ใน Octet สุดท้าย
```

ตัวอย่าง `/27`

```text
Subnet Mask = 255.255.255.224
Block Size   = 256 - 224
             = 32
```

Network Address ของ `/27` ใน `/24` จึงเพิ่มครั้งละ 32 เช่น `.0`, `.32`, `.64`, `.96` และต่อไป

### 5.7 ขั้นตอนออกแบบ VLSM

```text
1. เรียงความต้องการ Host จากมากไปน้อย
2. หา Host Bits ที่รองรับแต่ละความต้องการ
3. แปลงเป็น Prefix และ Subnet Mask
4. จัดสรร Subnet ใหญ่ที่สุดก่อน
5. เริ่ม Subnet ถัดไปหลัง Broadcast Address ของ Subnet ก่อนหน้า
6. ตรวจช่วงไม่ให้ซ้อนทับกัน
7. กำหนด Gateway และ Host Address
```

| หน่วยงาน | Host ที่ต้องการ | Network/Prefix | Subnet Mask | Usable Host Range | Broadcast |

### แนวคำตอบคำถามท้ายแล็บ

1. FLSM ใช้ Mask เดียวกันทุก Subnet ส่วน VLSM เลือก Mask ต่างกันตามจำนวน Host
2. เพื่อวาง Subnet ขนาดใหญ่ก่อน ลดโอกาสที่ Address Space จะแตกเป็นช่วงเล็กจนจัดสรร Subnet ใหญ่ไม่ได้
3. 6 Host Bits รองรับ 62 Hosts และได้ `/26`
4. `255.255.255.224` และมี 30 Usable Hosts
5. 16 คำนวณจาก `256 - 240`
6. Network `192.168.100.64` และ Broadcast `192.168.100.95`
7. `.96` เป็น Network Address และ `.111` เป็น Broadcast Address ของ `/28`
8. ทั้งสาม Subnet เป็น Directly Connected Network ของ R1
9. `/26` ทำให้ SALES-PC คำนวณ `.100` ว่าอยู่ใน Local Subnet `.64/26` จึงทำ ARP หาโดยตรง
10. `192.168.100.116`
