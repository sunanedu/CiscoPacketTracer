# ใบงานหน่วยที่ 1 แล็บที่ 1

## การเริ่มต้นใช้งาน Cisco Packet Tracer และรู้จักส่วนประกอบของโปรแกรม

| รายการ | รายละเอียด |
|---|---|
| รหัสใบงาน | CPT-U01-L01 |
| ระดับ | ผู้เริ่มต้น |
| ระยะเวลาโดยประมาณ | 90 นาที |
| รูปแบบการทำงาน | รายบุคคล หรือกลุ่มละไม่เกิน 2 คน |
| ไฟล์ที่ต้องส่ง | `CPT-U01-L01-ชื่อผู้เรียน.pkt` และใบงานที่บันทึกผลแล้ว |

---

## 1. เรื่องของใบงาน

ใบงานนี้เป็นการทำความรู้จักโปรแกรม Cisco Packet Tracer ตั้งแต่การสร้างไฟล์งานใหม่ การเลือกและวางอุปกรณ์ การย้าย ลบ และเปลี่ยนชื่ออุปกรณ์ การต่อสายเบื้องต้น การสังเกตสถานะของพอร์ต การสลับระหว่างโหมด **Realtime** และ **Simulation** ตลอดจนการบันทึกและเปิดไฟล์งาน

แล็บนี้มุ่งเน้นการใช้งานโปรแกรม จึง **ยังไม่มีการกำหนด IP Address และยังไม่มีคำสั่ง Cisco IOS** ผู้เรียนจะได้ฝึกเรื่องดังกล่าวในแล็บถัดไป

## 2. เป้าหมายการเรียนรู้

เมื่อทำใบงานเสร็จ ผู้เรียนควรสามารถทำสิ่งต่อไปนี้ได้

1. เปิดโปรแกรมและสร้างไฟล์ Packet Tracer ใหม่ได้
2. อธิบายหน้าที่ของพื้นที่ทำงานและแถบเครื่องมือหลักได้
3. เลือก วาง ย้าย ลบ และเปลี่ยนชื่ออุปกรณ์ได้
4. วาง PC และ Switch ตามแผนผังที่กำหนดได้
5. ต่อสาย Copper Straight-Through เข้ากับพอร์ตที่ถูกต้องได้
6. ตรวจสอบสถานะการเชื่อมต่อจากสัญลักษณ์และไฟสถานะของพอร์ตได้
7. สลับระหว่าง Realtime Mode และ Simulation Mode ได้
8. บันทึก ปิด และเปิดไฟล์ `.pkt` กลับมาตรวจสอบได้

## 3. ความรู้พื้นฐานก่อนเริ่ม

ผู้เรียนไม่จำเป็นต้องมีพื้นฐานระบบเครือข่ายมาก่อน แต่ควรใช้งานเมาส์ คีย์บอร์ด และจัดการไฟล์บนระบบปฏิบัติการได้

## 4. อุปกรณ์และซอฟต์แวร์ที่ใช้

### 4.1 ซอฟต์แวร์

- โปรแกรม Cisco Packet Tracer จำนวน 1 ชุด
- โฟลเดอร์สำหรับบันทึกไฟล์งานของผู้เรียน

### 4.2 อุปกรณ์จำลองใน Packet Tracer

| ลำดับ | ประเภทอุปกรณ์ | รุ่นที่แนะนำ | จำนวน | ชื่อที่กำหนด |
|---:|---|---|---:|---|
| 1 | Switch | Cisco 2960 | 1 | `SW1` |
| 2 | PC | PC-PT | 1 | `PC1` |
| 3 | PC | PC-PT | 1 | `PC2` |

> หมายเหตุ: หากโปรแกรมรุ่นที่ใช้งานแสดงชื่อรุ่นแตกต่างเล็กน้อย ให้เลือก Switch ตระกูล 2960 และอุปกรณ์ปลายทางชนิด PC

## 6. ตารางกำหนด IP Address

ในแล็บนี้ยังไม่มีการกำหนด IP Address, Subnet Mask และ Default Gateway

| อุปกรณ์ | พอร์ต | IP Address | Subnet Mask | Default Gateway |
|---|---|---|---|---|
| PC1 | FastEthernet0 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |
| PC2 | FastEthernet0 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |
| SW1 | VLAN 1 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |

> การที่ยังไม่มี IP Address หมายความว่าอุปกรณ์อาจเชื่อมต่อกันทางกายภาพได้ แต่ยังไม่พร้อมทดสอบการสื่อสารด้วยคำสั่ง `ping`

## 8. รู้จักส่วนประกอบหลักของหน้าจอ

ก่อนลงมือ ให้ผู้เรียนค้นหาส่วนประกอบต่อไปนี้บนหน้าจอของตนเอง ตำแหน่งอาจแตกต่างเล็กน้อยตามรุ่นของโปรแกรม

| ส่วนประกอบ | หน้าที่ |
|---|---|
| Menu Bar | สร้าง เปิด บันทึก และจัดการไฟล์งาน |
| Main Toolbar | รวมปุ่มคำสั่งที่ใช้บ่อย เช่น New, Open, Save, Undo และ Redo |
| Logical Workspace | พื้นที่หลักสำหรับสร้างและจัดวางแผนผังเครือข่าย |
| Physical Workspace | แสดงอุปกรณ์ตามสถานที่จริง เช่น เมือง อาคาร และห้อง |

| Simulation Mode | ใช้ติดตามเหตุการณ์และการเดินทางของ Packet ทีละขั้นตอน |

## 9. ขั้นตอนการปฏิบัติงาน

### ขั้นตอนที่ 1: สร้างไฟล์งานใหม่

1. เปิดโปรแกรม Cisco Packet Tracer
2. เลือกเมนู **File > New** หรือกดปุ่ม New บนแถบเครื่องมือ
3. ตรวจสอบว่าพื้นที่ทำงานว่างและอยู่ในมุมมอง **Logical**
4. เลือก **File > Save As**
5. บันทึกไฟล์ชื่อ `CPT-U01-L01-ชื่อผู้เรียน.pkt`

> ควรบันทึกไฟล์ตั้งแต่เริ่มงาน และกด `Ctrl+S` เป็นระยะเพื่อลดความเสี่ยงจากการสูญหายของงาน

### ขั้นตอนที่ 2: วาง Switch

1. ที่ส่วนเลือกอุปกรณ์ เลือก **Network Devices**
2. เลือกหมวดย่อย **Switches**
3. เลือก Switch รุ่น **2960**
4. คลิกพื้นที่ทำงานหนึ่งครั้งเพื่อวาง Switch ไว้บริเวณกึ่งกลางด้านบน
5. คลิกอุปกรณ์และเปลี่ยน **Display Name** จากชื่อเดิมเป็น `SW1`

### ขั้นตอนที่ 3: วางเครื่อง PC

1. เลือก **End Devices**
2. เลือกอุปกรณ์ **PC** จำนวน 2 เครื่อง
3. วาง PC เครื่องแรกด้านล่างซ้ายของ SW1
4. วาง PC เครื่องที่สองด้านล่างขวาของ SW1
5. เปลี่ยน Display Name ของเครื่องแรกเป็น `PC1`
6. เปลี่ยน Display Name ของเครื่องที่สองเป็น `PC2`

### ขั้นตอนที่ 4: ฝึกเลือก ย้าย และยกเลิกคำสั่ง

1. ใช้เครื่องมือ Select คลิกเลือก PC1
2. ลาก PC1 ไปยังตำแหน่งใหม่เล็กน้อย
3. ทดลองกด Undo แล้วสังเกตผล
4. ทดลองกด Redo แล้วสังเกตผล
5. จัดวางอุปกรณ์ให้เป็นระเบียบตามแผนผังในข้อ 5

> ไม่จำเป็นต้องลบอุปกรณ์จริงในไฟล์งาน หากต้องการทดลองเครื่องมือ Delete ให้เพิ่ม PC ชั่วคราวอีกหนึ่งเครื่อง ลบ PC ชั่วคราวนั้น แล้วตรวจสอบว่า PC1, PC2 และ SW1 ยังอยู่ครบ

### ขั้นตอนที่ 5: ต่อสาย PC1 เข้ากับ SW1

1. เลือกหมวด **Connections** ซึ่งใช้สัญลักษณ์รูปสายหรือสายฟ้า
2. เลือกสาย **Copper Straight-Through**
3. คลิก `PC1` และเลือกพอร์ต `FastEthernet0`
4. คลิก `SW1` และเลือกพอร์ต `FastEthernet0/1`
5. ตรวจสอบว่าเกิดเส้นสายเชื่อมระหว่าง PC1 และ SW1

### ขั้นตอนที่ 6: ต่อสาย PC2 เข้ากับ SW1

1. เลือกสาย **Copper Straight-Through** อีกครั้ง
2. คลิก `PC2` และเลือกพอร์ต `FastEthernet0`
3. คลิก `SW1` และเลือกพอร์ต `FastEthernet0/2`
4. ตรวจสอบว่าเกิดเส้นสายเชื่อมระหว่าง PC2 และ SW1
5. รอให้ไฟสถานะที่ปลายสายเปลี่ยนเป็นสีเขียว ซึ่งอาจใช้เวลาสักครู่

> หากไฟฝั่ง Switch แสดงสีส้มหรือสีเหลืองชั่วคราวหลังต่อสาย อาจเป็นกระบวนการเริ่มทำงานตามปกติของ Switch ให้รอสักครู่ก่อนวินิจฉัยว่าเกิดข้อผิดพลาด

### ขั้นตอนที่ 7: สำรวจหน้าต่างของอุปกรณ์

1. คลิก `PC1` และสำรวจแท็บ **Physical**, **Config** และ **Desktop**
2. ในแท็บ Desktop ให้ค้นหาเครื่องมือ **IP Configuration** และ **Command Prompt** แต่ยังไม่ต้องเปลี่ยนค่าใด
3. ปิดหน้าต่าง PC1
4. คลิก `SW1` และสำรวจแท็บ **Physical**, **Config** และ **CLI**
5. ยังไม่ต้องพิมพ์คำสั่งหรือเปลี่ยนค่าของ SW1
6. ปิดหน้าต่าง SW1

### ขั้นตอนที่ 8: ทดลอง Realtime และ Simulation Mode

1. ตรวจสอบว่าขณะนี้โปรแกรมอยู่ใน **Realtime Mode**
2. คลิกเปลี่ยนเป็น **Simulation Mode**
3. สังเกตแผง Simulation Panel และ Event List
4. ค้นหาปุ่มควบคุม เช่น Play, Capture/Forward และ Reset Simulation
5. ยังไม่ต้องสร้าง Packet เพราะอุปกรณ์ยังไม่ได้กำหนด IP Address
6. เปลี่ยนกลับเป็น **Realtime Mode**

### ขั้นตอนที่ 9: บันทึกและเปิดไฟล์เพื่อตรวจสอบ

1. กด `Ctrl+S` เพื่อบันทึกงาน
2. ตรวจสอบว่าไฟล์มีนามสกุล `.pkt`
3. ปิดไฟล์งานหรือปิดโปรแกรม
4. เปิดไฟล์ `CPT-U01-L01-ชื่อผู้เรียน.pkt` อีกครั้ง
5. ตรวจสอบว่าอุปกรณ์ ชื่ออุปกรณ์ และสายสัญญาณยังอยู่ครบ

## 10. คำสั่งที่ใช้ในแล็บ

แล็บนี้ไม่มีคำสั่ง Cisco IOS เนื่องจากมีเป้าหมายให้ผู้เรียนคุ้นเคยกับหน้าจอและเครื่องมือของโปรแกรมก่อน

คำสั่งลัดของโปรแกรมที่ควรรู้มีดังนี้

| คำสั่งลัด | หน้าที่ |
|---|---|
| `Ctrl+N` | สร้างไฟล์งานใหม่ |
| `Ctrl+O` | เปิดไฟล์งาน |
| `Ctrl+S` | บันทึกไฟล์งาน |
| `Ctrl+Z` | ยกเลิกการกระทำล่าสุด |
| `Ctrl+Y` | ทำซ้ำการกระทำที่ยกเลิก |

## ภาคผนวกสำหรับผู้สอน: แนวคำตอบย่อ

1. **Logical Workspace** ใช้สร้างความสัมพันธ์เชิงตรรกะและแผนผังเครือข่าย ส่วน **Physical Workspace** ใช้แสดงตำแหน่งทางกายภาพของอุปกรณ์
2. ยังไม่ได้กำหนด IP Address และ Subnet Mask ให้ PC1 และ PC2
3. Copper Straight-Through
4. Realtime เหมาะกับการใช้งานและตรวจสอบเครือข่ายตามปกติ ส่วน Simulation เหมาะกับการติดตามเหตุการณ์และวิเคราะห์ Packet ทีละขั้นตอน
5. รอสักครู่เพื่อให้พอร์ตของ Switch เริ่มทำงาน แล้วจึงตรวจสอบสายและพอร์ตหากสถานะยังไม่เปลี่ยนเป็นสีเขียว
