## 1. เรื่องของใบงาน

ใบงานนี้เป็นการเรียนรู้อุปกรณ์พื้นฐานที่ใช้สร้างระบบเครือข่าย ได้แก่ PC, Server, Switch, Router และ Access Point ผู้เรียนจะได้สำรวจพอร์ตของอุปกรณ์แต่ละชนิด และฝึกเลือกสายสัญญาณให้เหมาะกับอุปกรณ์ต้นทางและปลายทาง

ผู้เรียนจะสร้างแผนผังเครือข่ายตัวอย่างที่มีทั้งสาย **Copper Straight-Through**, **Copper Crossover** และ **Console Cable** พร้อมตรวจสอบชื่อพอร์ตและสถานะการเชื่อมต่อ

แล็บนี้เน้นการเชื่อมต่อทางกายภาพ จึงยังไม่มีการกำหนด IP Address และยังไม่ใช้คำสั่ง Cisco IOS เพื่อกำหนดค่าอุปกรณ์

## 2. เป้าหมายการเรียนรู้

เมื่อทำใบงานเสร็จ ผู้เรียนควรสามารถทำสิ่งต่อไปนี้ได้

1. แยกอุปกรณ์ประเภท End Device และ Network Device ได้
2. อธิบายหน้าที่พื้นฐานของ PC, Server, Switch, Router และ Access Point ได้
3. ระบุพอร์ต FastEthernet, GigabitEthernet, RS-232 และ Console ได้
4. เลือก Copper Straight-Through สำหรับเชื่อมอุปกรณ์ต่างประเภทกันได้
5. เลือก Copper Crossover สำหรับเชื่อมอุปกรณ์ Ethernet ประเภทเดียวกันได้
6. ใช้ Console Cable เชื่อมเครื่องผู้ดูแลระบบเข้ากับ Router ได้
7. อธิบายกรณีใช้งานพื้นฐานของสาย Fiber ได้
8. ตรวจสอบสถานะลิงก์และแก้ไขการต่อสายผิดพอร์ตได้
9. จัดทำตารางบันทึกการเชื่อมต่อของเครือข่ายได้

## 3. ความรู้พื้นฐานก่อนเริ่ม

ผู้เรียนควรสามารถ

- เปิดและบันทึกไฟล์ Cisco Packet Tracer ได้
- เลือก วาง ย้าย ลบ และเปลี่ยนชื่ออุปกรณ์ได้
- สลับระหว่าง Realtime Mode และ Simulation Mode ได้

## 4. แนวคิดพื้นฐาน

### 4.1 End Device คืออะไร

End Device คืออุปกรณ์ต้นทางหรือปลายทางที่ผู้ใช้หรือบริการเครือข่ายใช้งาน เช่น PC, Laptop, Server, Printer, IP Phone และ Smartphone

### 4.2 Network Device คืออะไร

Network Device คืออุปกรณ์ที่ช่วยเชื่อมต่อ ควบคุม หรือส่งต่อข้อมูลในเครือข่าย เช่น Switch, Router, Access Point, Firewall และ Modem

### 4.3 หน้าที่ของอุปกรณ์ในแล็บ

| อุปกรณ์ | ประเภท | หน้าที่พื้นฐาน |
|---|---|---|
| PC | End Device | ใช้งานเครือข่ายและเป็นต้นทางหรือปลายทางของข้อมูล |
| Server | End Device | ให้บริการแก่เครื่องลูกข่าย เช่น Web, DNS, DHCP และ File Service |
| Switch | Network Device | เชื่อมอุปกรณ์ภายใน LAN และส่งต่อ Frame จาก MAC Address |
| Router | Network Device | เชื่อมเครือข่ายที่ใช้หมายเลข Network ต่างกันและเลือกเส้นทางของ Packet |
| Access Point | Network Device | เชื่อมอุปกรณ์ไร้สายเข้ากับเครือข่ายแบบใช้สาย |

### 4.4 ชื่อพอร์ตที่ควรรู้

| ชื่อพอร์ต | ตัวอย่างชื่อย่อ | การใช้งานทั่วไป |
|---|---|---|
| FastEthernet | `Fa0`, `Fa0/1` | Ethernet ความเร็วสูงสุด 100 Mbps |
| GigabitEthernet | `Gi0/0`, `Gi0/1` | Ethernet ความเร็วสูงสุด 1 Gbps |
| RS-232 | `RS-232` | พอร์ต Serial บน PC สำหรับเชื่อมต่อ Console |
| Console | `Console` | ใช้บริหาร Router หรือ Switch โดยตรง ไม่ใช่พอร์ตส่งข้อมูล LAN |
| Port 0 | `Port 0` | ชื่อพอร์ต Ethernet ที่พบใน Access Point บางรุ่นใน Packet Tracer |

> รูปแบบชื่อพอร์ตขึ้นอยู่กับชนิดและรุ่นของอุปกรณ์ ตัวอย่างเช่น `FastEthernet0` เป็นพอร์ตของ PC ส่วน `FastEthernet0/1` เป็นพอร์ตหมายเลข 1 บน Switch

## 5. หลักการเลือกสายสัญญาณเบื้องต้น

| สายสัญญาณ | ใช้เชื่อมต่อโดยทั่วไป | ตัวอย่างในแล็บ |
|---|---|---|
| Copper Straight-Through | อุปกรณ์ Ethernet ต่างประเภทกัน | PC–Switch, Server–Switch, Router–Switch, AP–Switch |
| Copper Crossover | อุปกรณ์ Ethernet ประเภทเดียวกัน | Switch–Switch, PC–PC, Router–Router |
| Console | เครื่องผู้ดูแลเข้ากับพอร์ต Console ของอุปกรณ์เครือข่าย | PC RS-232–Router Console |
| Fiber | อุปกรณ์ที่มีพอร์ต Fiber ชนิดที่เข้ากันได้ | Switch–Switch ระยะไกลหรือ Uplink ความเร็วสูง |
| Automatically Choose Connection Type | ให้ Packet Tracer เลือกสายและพอร์ตให้อัตโนมัติ | ใช้ตรวจสอบตนเองหลังฝึกเลือกสายด้วยมือแล้ว |

> กฎ Straight-Through/Crossover ในตารางเป็นหลักการพื้นฐานที่ใช้ในการเรียนรู้ อุปกรณ์จริงรุ่นใหม่บางรุ่นรองรับ Auto-MDI/MDIX และอาจปรับการรับส่งสัญญาณให้เองได้ อย่างไรก็ตาม ผู้เรียนควรเลือกสายตามที่กำหนดในใบงานนี้

## 6. อุปกรณ์และซอฟต์แวร์ที่ใช้

### 6.1 ซอฟต์แวร์

- โปรแกรม Cisco Packet Tracer จำนวน 1 ชุด
- โฟลเดอร์สำหรับบันทึกไฟล์งานของผู้เรียน

### 6.2 อุปกรณ์จำลองใน Packet Tracer

| ลำดับ | ประเภทอุปกรณ์ | รุ่นที่แนะนำ | จำนวน | ชื่อที่กำหนด |
|---:|---|---|---:|---|
| 1 | Router | Cisco 2911 | 1 | `R1` |
| 2 | Switch | Cisco 2960 | 2 | `SW1`, `SW2` |
| 3 | PC | PC-PT | 2 | `PC1`, `Admin-PC` |
| 4 | Server | Server-PT | 1 | `Server1` |
| 5 | Access Point | AccessPoint-PT | 1 | `AP1` |

> หากโปรแกรมรุ่นที่ใช้งานแสดงชื่อรุ่นต่างออกไปเล็กน้อย ให้เลือกอุปกรณ์ที่มีพอร์ตตามตารางการเชื่อมต่อในข้อ 9

## 8. ตารางกำหนด IP Address

ในแล็บนี้ยังไม่มีการกำหนด IP Address, Subnet Mask และ Default Gateway

| อุปกรณ์ | พอร์ตหรือ Interface | IP Address | Subnet Mask | Default Gateway |
|---|---|---|---|---|
| PC1 | FastEthernet0 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |
| Admin-PC | FastEthernet0 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |
| Server1 | FastEthernet0 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |
| R1 | GigabitEthernet0/0 | ยังไม่กำหนด | ยังไม่กำหนด | ไม่เกี่ยวข้อง |
| SW1 | VLAN 1 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |
| SW2 | VLAN 1 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |
| AP1 | Port 0 | ยังไม่กำหนด | ยังไม่กำหนด | ยังไม่กำหนด |

> แล็บนี้ทดสอบเฉพาะการวางอุปกรณ์ การเลือกพอร์ต และการต่อสาย จึงยังไม่ใช้ `ping` ทดสอบการรับส่งข้อมูล

## 10. ขั้นตอนการปฏิบัติงาน

### ตอนที่ 1: สร้างและบันทึกไฟล์งาน

1. เปิดโปรแกรม Cisco Packet Tracer
2. เลือก **File > New**
3. ตรวจสอบว่าอยู่ใน **Logical Workspace**
4. เลือก **File > Save As**
5. บันทึกไฟล์ชื่อ `CPT-U01-L02-ชื่อผู้เรียน.pkt`

### ตอนที่ 2: วาง Router และ Switch

1. เลือก **Network Devices > Routers**
2. เลือก Router รุ่น **2911** แล้ววางบนพื้นที่ทำงาน
3. เปลี่ยน Display Name เป็น `R1`
4. เลือก **Network Devices > Switches**
5. เลือก Switch รุ่น **2960** จำนวน 2 ตัว
6. เปลี่ยน Display Name เป็น `SW1` และ `SW2`
7. จัดวาง R1, SW1 และ SW2 ตามแผนผังในข้อ 7

### ตอนที่ 3: วาง End Device และ Access Point

1. เลือก **End Devices** แล้ววาง PC จำนวน 2 เครื่อง
2. เปลี่ยนชื่อเป็น `PC1` และ `Admin-PC`
3. วาง Server จำนวน 1 เครื่อง แล้วเปลี่ยนชื่อเป็น `Server1`
4. ค้นหา Access Point ในหมวด **Network Devices > Wireless Devices**
5. วาง Access Point จำนวน 1 ตัว แล้วเปลี่ยนชื่อเป็น `AP1`
6. จัดวางอุปกรณ์ทั้งหมดให้ตรงตามแผนผัง

### ตอนที่ 4: สำรวจอุปกรณ์และพอร์ต

ให้คลิกอุปกรณ์แต่ละตัวและสำรวจแท็บ Physical หรือ Config โดยยังไม่เปลี่ยนค่า จากนั้นบันทึกสิ่งที่พบ

| อุปกรณ์ | พอร์ตที่พบอย่างน้อย 1 พอร์ต | อุปกรณ์นี้เป็น End/Network Device |
|---|---|---|
| PC1 | ........................................ | ........................................ |
| Admin-PC | ........................................ | ........................................ |
| Server1 | ........................................ | ........................................ |
| R1 | ........................................ | ........................................ |
| SW1 | ........................................ | ........................................ |
| SW2 | ........................................ | ........................................ |
| AP1 | ........................................ | ........................................ |

คำแนะนำในการสำรวจ

1. คลิก `PC1` และหา `FastEthernet0`
2. คลิก `Admin-PC` และหา `RS-232`
3. คลิก `R1` และหา `GigabitEthernet0/0` กับ `Console`
4. คลิก `SW1` และหา `FastEthernet0/1`, `GigabitEthernet0/1` และ `GigabitEthernet0/2`
5. คลิก `AP1` และหาพอร์ต Ethernet ที่แสดงเป็น `Port 0` หรือชื่อใกล้เคียง

### ตอนที่ 5: เชื่อม PC1 เข้ากับ SW1

1. เลือก **Connections**
2. เลือก **Copper Straight-Through**
3. คลิก `PC1` แล้วเลือก `FastEthernet0`
4. คลิก `SW1` แล้วเลือก `FastEthernet0/1`
5. ตรวจสอบว่าเกิดเส้นเชื่อมระหว่างอุปกรณ์

เหตุผลที่เลือกสาย: PC เป็น End Device ส่วน Switch เป็น Network Device ซึ่งเป็นอุปกรณ์ Ethernet ต่างประเภทกัน

### ตอนที่ 6: เชื่อม Server1 เข้ากับ SW1

1. เลือก **Copper Straight-Through**
2. คลิก `Server1` แล้วเลือก `FastEthernet0`
3. คลิก `SW1` แล้วเลือก `FastEthernet0/2`
4. ตรวจสอบสายและชื่อพอร์ตอีกครั้ง

### ตอนที่ 7: เชื่อม R1 เข้ากับ SW1

1. เลือก **Copper Straight-Through**
2. คลิก `R1` แล้วเลือก `GigabitEthernet0/0`
3. คลิก `SW1` แล้วเลือก `GigabitEthernet0/1`
4. สังเกตสถานะที่ปลายสาย

> พอร์ต Ethernet ของ Router อาจยังไม่เป็นสีเขียว แม้เลือกสายและพอร์ตถูกต้อง เพราะ Interface ของ Router มักถูกปิดไว้ในค่าเริ่มต้น การเปิด Interface ด้วยคำสั่ง `no shutdown` จะเรียนในแล็บ Cisco IOS

### ตอนที่ 8: เชื่อม SW1 เข้ากับ SW2

1. เลือก **Copper Crossover**
2. คลิก `SW1` แล้วเลือก `GigabitEthernet0/2`
3. คลิก `SW2` แล้วเลือก `GigabitEthernet0/1`
4. รอให้พอร์ตของ Switch เริ่มทำงาน
5. ตรวจสอบว่าสายเส้นนี้เป็น Crossover ไม่ใช่ Straight-Through

เหตุผลที่เลือกสาย: เป็นการเชื่อมอุปกรณ์ Ethernet ประเภทเดียวกัน คือ Switch กับ Switch

### ตอนที่ 9: เชื่อม AP1 เข้ากับ SW2

1. เลือก **Copper Straight-Through**
2. คลิก `AP1` แล้วเลือก `Port 0` หรือพอร์ต Ethernet ที่โปรแกรมแสดง
3. คลิก `SW2` แล้วเลือก `FastEthernet0/1`
4. ตรวจสอบสถานะของลิงก์หลังรอให้อุปกรณ์เริ่มทำงาน

### ตอนที่ 10: เชื่อม Admin-PC เข้ากับ Console ของ R1

1. เลือก **Console Cable** ซึ่งโดยทั่วไปแสดงเป็นสายสีฟ้าอ่อน
2. คลิก `Admin-PC` แล้วเลือก `RS-232`
3. คลิก `R1` แล้วเลือก `Console`
4. คลิก `Admin-PC`
5. เปิดแท็บ **Desktop > Terminal**
6. ตรวจสอบว่าหน้าต่าง Terminal เปิดได้ โดยใช้ค่าการเชื่อมต่อเริ่มต้น
7. ยังไม่ต้องกำหนดค่า Router หรือพิมพ์คำสั่งใด

> Console Cable ใช้บริหารอุปกรณ์โดยตรง จึงไม่ต้องกำหนด IP Address ก่อนเปิด Terminal และไม่มีไฟสถานะลิงก์แบบสาย Ethernet

### ตอนที่ 11: สำรวจสาย Fiber

1. เปิดหมวด **Connections**
2. ค้นหาไอคอนสาย **Fiber**
3. เลื่อนตัวชี้เมาส์เหนือไอคอนเพื่อดูชื่อ
4. ไม่ต้องนำสาย Fiber ไปต่อกับพอร์ต Copper ในแผนผังนี้

## ภาคผนวกสำหรับผู้สอน: แนวคำตอบย่อ

### แนวคำตอบแบบฝึกจำแนกสาย

| การเชื่อมต่อ | แนวคำตอบ |
|---|---|
| PC กับ Switch | Copper Straight-Through |
| Router กับ Switch | Copper Straight-Through |
| Switch กับ Switch | Copper Crossover |
| PC กับ PC ผ่านพอร์ต Ethernet โดยตรง | Copper Crossover |
| PC RS-232 กับ Router Console | Console Cable |
| พอร์ต Fiber ของ Switch กับพอร์ต Fiber ของ Switch | Fiber |

### แนวคำตอบคำถามท้ายแล็บ

1. Switch เชื่อมอุปกรณ์ใน LAN และส่งต่อ Frame จาก MAC Address ส่วน Router เชื่อมเครือข่ายที่ต่างกันและส่งต่อ Packet จาก IP Address
2. PC และ Switch เป็นอุปกรณ์ Ethernet ต่างประเภทกัน ตามหลักการพื้นฐานจึงใช้ Straight-Through
3. Copper Crossover
4. พอร์ต Console
5. ไม่ต้องกำหนด IP เพราะ Console เป็นการเชื่อมต่อบริหารอุปกรณ์โดยตรงแบบ Out-of-Band
6. สื่อส่งสัญญาณและหัวต่อคนละชนิด พอร์ต RJ-45 ใช้สัญญาณไฟฟ้าผ่านสายทองแดง ส่วน Fiber ใช้แสงและต้องใช้พอร์ตหรือ Module ที่รองรับ
7. ตรวจสอบว่า Interface ของ Router อยู่ในสถานะ Shutdown หรือไม่
8. เครื่องมืออัตโนมัติช่วยเลือกสายและพอร์ตได้รวดเร็ว แต่การเลือกด้วยตนเองทำให้เข้าใจชนิดอุปกรณ์ พอร์ต และหลักการเดินสาย ซึ่งจำเป็นต่อการแก้ปัญหา

### ข้อสังเกตสำหรับผู้สอน

- ไม่ควรหักคะแนนผู้เรียนเพียงเพราะพอร์ต Router ยังเป็นสีแดง หากสายและพอร์ตถูกต้อง เนื่องจาก Interface อาจอยู่ในสถานะ Administratively Down
- อุปกรณ์จริงรุ่นใหม่อาจรองรับ Auto-MDI/MDIX แต่แล็บนี้ใช้หลักการเลือกสายแบบดั้งเดิมเพื่อสร้างความเข้าใจพื้นฐาน
- หากชื่อพอร์ตของ Access Point ต่างจาก `Port 0` ให้ยอมรับพอร์ต Ethernet ที่ตรงกับรุ่นซึ่งผู้เรียนใช้งาน
