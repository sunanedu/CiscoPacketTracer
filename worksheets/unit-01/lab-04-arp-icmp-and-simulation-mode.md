## 1. เรื่องของใบงาน

ใบงานนี้ใช้ Simulation Mode ของ Cisco Packet Tracer เพื่อติดตามการทำงานของ **ARP** และ **ICMP** ภายในเครือข่าย LAN ผู้เรียนจะสังเกตตั้งแต่ PC1 ค้นหา MAC Address ของ PC2 ไปจนถึงการส่ง ICMP Echo Request และการรับ ICMP Echo Reply

ผู้เรียนจะทดลองสองสถานการณ์ ได้แก่ การส่งข้อมูลเมื่อ ARP Cache ยังไม่มีข้อมูล และการส่งข้อมูลซ้ำเมื่อ ARP Cache มีข้อมูลของปลายทางแล้ว เพื่อเปรียบเทียบจำนวนและชนิดของ Packet ที่เกิดขึ้น

## 2. เป้าหมายการเรียนรู้

เมื่อทำใบงานเสร็จ ผู้เรียนควรสามารถทำสิ่งต่อไปนี้ได้

1. อธิบายหน้าที่พื้นฐานของ ARP และ ICMP ได้
2. อธิบายความสัมพันธ์ระหว่าง IPv4 Address และ MAC Address ได้
3. ใช้ `arp -a` ตรวจสอบ ARP Cache ของ PC ได้
4. ล้าง ARP Cache เพื่อสร้างสถานการณ์ทดสอบใหม่ได้
5. ตั้งค่า Event List Filters ให้แสดงเฉพาะ ARP และ ICMP ได้
6. ใช้ Add Simple PDU และ Capture/Forward ใน Simulation Mode ได้
7. ระบุ ARP Request, ARP Reply, ICMP Echo Request และ ICMP Echo Reply ได้
8. อธิบายเหตุผลที่ ARP Request ใช้ Broadcast ได้
9. เปรียบเทียบเหตุการณ์เมื่อ ARP Cache ว่างและเมื่อมีข้อมูลแล้วได้
10. ใช้ข้อมูลจาก Event List และ PDU Details วิเคราะห์ปัญหาเบื้องต้นได้

## 4. แนวคิดพื้นฐาน

### 4.1 IP Address และ MAC Address ต่างกันอย่างไร

| รายการ | IP Address | MAC Address |
|---|---|---|
| หน้าที่ | ระบุตำแหน่งทางตรรกะของอุปกรณ์ในเครือข่าย | ระบุ Network Interface ภายใน LAN |
| ตัวอย่าง | `192.168.10.20` | `00D0.BA12.3456` หรือรูปแบบใกล้เคียง |
| ชั้นโดยทั่วไป | Network Layer หรือ Layer 3 | Data Link Layer หรือ Layer 2 |
| การกำหนดค่า | ผู้ดูแลหรือ DHCP สามารถกำหนดได้ | ผูกกับ Network Interface และ Packet Tracer สร้างให้ |

> ค่า MAC Address ในเครื่องของผู้เรียนจะไม่จำเป็นต้องตรงกับตัวอย่าง เพราะ Packet Tracer สร้างค่าให้กับอุปกรณ์แต่ละตัว

### 4.2 ARP คืออะไร

ARP ย่อมาจาก **Address Resolution Protocol** ใช้ค้นหาว่า IPv4 Address ของอุปกรณ์ภายใน LAN เดียวกันสัมพันธ์กับ MAC Address ใด

ตัวอย่างคำถามของ PC1 ในแล็บนี้

```text
ใครมี IP Address 192.168.10.20 ให้แจ้ง MAC Address กลับมายัง 192.168.10.10
```

### 4.3 ICMP คืออะไร

ICMP ย่อมาจาก **Internet Control Message Protocol** ใช้ส่งข้อความควบคุมและรายงานสถานะของเครือข่าย คำสั่ง `ping` ใช้ ICMP สองข้อความสำคัญ

- **ICMP Echo Request** ส่งจากเครื่องต้นทางไปยังเครื่องปลายทาง
- **ICMP Echo Reply** ส่งจากเครื่องปลายทางกลับไปยังเครื่องต้นทาง

### 4.4 ARP Cache คืออะไร

ARP Cache เป็นตารางชั่วคราวที่เก็บความสัมพันธ์ระหว่าง IPv4 Address และ MAC Address เมื่อ PC1 รู้ MAC Address ของ PC2 แล้ว การส่งข้อมูลครั้งต่อไปอาจไม่ต้องส่ง ARP Request ใหม่ทันที

### 4.5 Broadcast และ Unicast

| ประเภท | ความหมาย | ตัวอย่างในแล็บ |
|---|---|---|
| Broadcast | ส่ง Frame ไปยังอุปกรณ์ทุกตัวใน Broadcast Domain | ARP Request |
| Unicast | ส่ง Frame ไปยังอุปกรณ์ปลายทางตัวเดียว | ARP Reply และ ICMP ที่ทราบ MAC ปลายทางแล้ว |

MAC Address ปลายทางสำหรับ Ethernet Broadcast คือ

```text
FFFF.FFFF.FFFF
```

โปรแกรมบางมุมมองอาจแสดงเป็น

```text
FF:FF:FF:FF:FF:FF
```

### 4.6 ลำดับเหตุการณ์ที่คาดหวัง

```text
PC1                          SW1                          PC2
 |                            |                            |
 |--- ARP Request Broadcast ->|--- ARP Request ---------->|
 |                            |                            |
 |<-- ARP Reply Unicast ------|<-- ARP Reply -------------|
 |                            |                            |
 |--- ICMP Echo Request ----->|--- ICMP Echo Request ---->|
 |                            |                            |
 |<-- ICMP Echo Reply --------|<-- ICMP Echo Reply -------|
 |                            |                            |
```

### ตอนที่ 5: ล้าง ARP Cache

ที่ Command Prompt ของ PC1 ให้ทดลองคำสั่ง

```text
arp -d
```

จากนั้นตรวจสอบอีกครั้ง

```text
arp -a
```

ผลที่คาดหวังคือรายการ Dynamic ARP ของ PC2 หายไป

> Packet Tracer บางรุ่นอาจรองรับรูปแบบคำสั่งแตกต่างกัน หาก `arp -d` ไม่ทำงาน ให้พิมพ์ `arp ?` หรือ `help arp` เพื่อตรวจรูปแบบที่รองรับ แล้วลบรายการของ `192.168.10.20` หากยังไม่สามารถลบได้ ให้ปิดไฟล์โดยไม่บันทึก เปิดใหม่ และหลีกเลี่ยงการ Ping ก่อนเริ่ม Simulation

## ภาคผนวกสำหรับผู้สอน: แนวคำตอบย่อ

### ลำดับเหตุการณ์

| ลำดับ | เหตุการณ์ | ต้นทาง | ปลายทาง | รูปแบบ | จุดประสงค์ |
|---:|---|---|---|---|---|
| 1 | ARP Request | PC1 | ทุกอุปกรณ์ใน Broadcast Domain | Broadcast | ค้นหา MAC ของ `192.168.10.20` |
| 2 | ARP Reply | PC2 | PC1 | Unicast | แจ้ง MAC Address ของ PC2 |
| 3 | ICMP Echo Request | PC1 | PC2 | Unicast | ขอทดสอบการเข้าถึง PC2 |
| 4 | ICMP Echo Reply | PC2 | PC1 | Unicast | ตอบกลับว่ารับ Echo Request แล้ว |

### แนวคำตอบคำถามท้ายแล็บ

1. ARP ใช้ค้นหา MAC Address ที่สัมพันธ์กับ IPv4 Address ภายใน LAN
2. การส่งข้อมูลผ่าน Ethernet ต้องมี Source และ Destination MAC Address ใน Frame
3. ใช้ Broadcast และ Destination MAC เป็น `FFFF.FFFF.FFFF`
4. ARP Request มีข้อมูล MAC Address ของ PC1 ทำให้ PC2 รู้ว่าจะส่ง Reply กลับไปที่ใด
5. ICMP Echo Request และ ICMP Echo Reply
6. แสดงตารางความสัมพันธ์ระหว่าง IPv4 Address และ MAC Address ที่เครื่องเรียนรู้ไว้
7. PC1 มี MAC ของ PC2 ใน ARP Cache แล้ว จึงสร้าง Ethernet Frame ได้โดยไม่ต้องค้นหาใหม่
8. Switch ส่งต่อหรือ Flood Broadcast ออกพอร์ตที่เกี่ยวข้องทั้งหมดใน VLAN เดียวกัน ยกเว้นพอร์ตที่รับ Frame เข้ามา
9. Source IP คือ `192.168.10.20` และ Destination IP คือ `192.168.10.10`
10. Simulation Mode แสดง Protocol ลำดับเวลา การเดินทางผ่านอุปกรณ์ และรายละเอียด Header ทำให้ระบุตำแหน่งหรือขั้นตอนที่ล้มเหลวได้

### ข้อสังเกตสำหรับผู้สอน

- จำนวน Event ในตารางอาจแตกต่างตามรุ่น การตั้งค่า และวิธีนับเหตุการณ์ที่ผ่าน Switch ควรประเมินจากชนิดและลำดับ Protocol มากกว่าจำนวนตายตัว
- หาก `arp -d` มี Syntax ต่างออกไป ให้ผู้เรียนใช้ Help ของ Command Prompt รุ่นนั้น และยอมรับวิธีอื่นที่ทำให้รายการ Dynamic ARP หายไปจริง
- ค่า MAC Address ของผู้เรียนแต่ละคนอาจไม่เหมือนกัน ควรตรวจความสัมพันธ์กับอุปกรณ์จริงในไฟล์ ไม่ตรวจจากค่าตายตัว
- หาก ARP Cache มีรายการจาก Realtime ping ตอนเริ่มแล็บ ต้องล้าง Cache ก่อนเริ่มสถานการณ์ที่ 1
