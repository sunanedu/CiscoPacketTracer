## 1. เรื่องของใบงาน

ใบงานนี้เป็นการสร้างเครือข่าย Local Area Network หรือ LAN แรกด้วย PC จำนวน 2 เครื่องและ Switch จำนวน 1 ตัว ผู้เรียนจะกำหนด IPv4 Address และ Subnet Mask แบบ Static ให้กับ PC แต่ละเครื่อง จากนั้นตรวจสอบค่าด้วยคำสั่ง `ipconfig` และทดสอบการสื่อสารด้วยคำสั่ง `ping`

แล็บนี้เป็นครั้งแรกที่อุปกรณ์จะสามารถรับส่งข้อมูลกันได้จริง ผู้เรียนจึงต้องตรวจสอบให้ครบทั้งการเชื่อมต่อทางกายภาพและการกำหนดค่าทางตรรกะ

## 2. เป้าหมายการเรียนรู้

เมื่อทำใบงานเสร็จ ผู้เรียนควรสามารถทำสิ่งต่อไปนี้ได้

1. สร้าง LAN ขนาดเล็กจาก PC และ Switch ได้
2. กำหนด IPv4 Address และ Subnet Mask แบบ Static ผ่านหน้าต่าง IP Configuration ได้
3. อธิบายความหมายเบื้องต้นของ IP Address, Subnet Mask และ Default Gateway ได้
4. ระบุ Network Address และช่วง Host ของเครือข่าย `/24` ได้
5. ใช้คำสั่ง `ipconfig` ตรวจสอบค่าของเครื่องได้
6. ใช้คำสั่ง `ping` ทดสอบ TCP/IP ของเครื่องตนเองและเครื่องปลายทางได้
7. แปลผล `ping` เบื้องต้นว่าเครือข่ายทำงานสำเร็จหรือล้มเหลวได้
8. แก้ไขปัญหาจาก IP Address ผิดเครือข่ายได้
9. บันทึกผลการทดสอบและสรุปผลการทำงานได้

## 3. ความรู้พื้นฐานก่อนเริ่ม

ผู้เรียนควรสามารถ

- เลือก วาง และเปลี่ยนชื่อ PC กับ Switch ได้
- ระบุพอร์ต FastEthernet ของ PC และ Switch ได้
- เลือกใช้สาย Copper Straight-Through ได้
- เปิด Command Prompt จากแท็บ Desktop ของ PC ได้

## 4. แนวคิดพื้นฐาน

### 4.1 LAN คืออะไร

LAN ย่อมาจาก **Local Area Network** เป็นเครือข่ายที่เชื่อมอุปกรณ์ภายในพื้นที่จำกัด เช่น ห้องเรียน ห้องปฏิบัติการ บ้าน หรือสำนักงาน ในแล็บนี้ PC1 และ PC2 อยู่ใน LAN เดียวกันและเชื่อมต่อผ่าน SW1

### 4.2 IP Address คืออะไร

IPv4 Address เป็นหมายเลขที่ใช้ระบุอุปกรณ์ในเครือข่าย ตัวอย่างในแล็บนี้คือ `192.168.10.10` และ `192.168.10.20` อุปกรณ์ที่อยู่ในเครือข่ายเดียวกันต้องมี IP Address ไม่ซ้ำกัน

### 4.3 Subnet Mask คืออะไร

Subnet Mask ใช้บอกว่าส่วนใดของ IP Address เป็นส่วนเครือข่ายและส่วนใดเป็นส่วน Host ในแล็บนี้ใช้ `255.255.255.0` หรือเขียนแบบ Prefix Length ได้เป็น `/24`

### 4.4 Default Gateway คืออะไร

Default Gateway คือ IP Address ของ Router ที่อุปกรณ์ใช้ส่งข้อมูลไปยังเครือข่ายอื่น แล็บนี้ไม่มี Router และ PC ทั้งสองเครื่องอยู่ใน LAN เดียวกัน จึงยังไม่ต้องกำหนด Default Gateway

### 4.5 ข้อมูลของเครือข่ายที่ใช้

| รายการ | ค่า |
|---|---|
| Network Address | `192.168.10.0` |
| Prefix Length | `/24` |
| Subnet Mask | `255.255.255.0` |
| ช่วง Host ที่กำหนดให้อุปกรณ์ได้ | `192.168.10.1` ถึง `192.168.10.254` |
| Broadcast Address | `192.168.10.255` |

> ห้ามกำหนด `192.168.10.0` หรือ `192.168.10.255` ให้กับ PC เพราะเป็น Network Address และ Broadcast Address ตามลำดับ

## 5. อุปกรณ์และซอฟต์แวร์ที่ใช้

### 5.1 ซอฟต์แวร์

- โปรแกรม Cisco Packet Tracer จำนวน 1 ชุด
- โฟลเดอร์สำหรับบันทึกไฟล์งานของผู้เรียน

### 5.2 อุปกรณ์จำลองใน Packet Tracer

| ลำดับ | ประเภทอุปกรณ์ | รุ่นที่แนะนำ | จำนวน | ชื่อที่กำหนด |
|---:|---|---|---:|---|
| 1 | Switch | Cisco 2960 | 1 | `SW1` |
| 2 | PC | PC-PT | 1 | `PC1` |
| 3 | PC | PC-PT | 1 | `PC2` |

## 7. ตารางกำหนด IP Address

| อุปกรณ์ | พอร์ต | IP Address | Subnet Mask | Default Gateway |
|---|---|---|---|---|
| PC1 | FastEthernet0 | `192.168.10.10` | `255.255.255.0` | ไม่กำหนด |
| PC2 | FastEthernet0 | `192.168.10.20` | `255.255.255.0` | ไม่กำหนด |

## 10. ขั้นตอนการปฏิบัติงาน

### ตอนที่ 1: สร้างและบันทึกไฟล์งาน

1. เปิดโปรแกรม Cisco Packet Tracer
2. เลือก **File > New**
3. ตรวจสอบว่าอยู่ใน **Logical Workspace**
4. เลือก **File > Save As**
5. บันทึกไฟล์ชื่อ `CPT-U01-L03-ชื่อผู้เรียน.pkt`

### ตอนที่ 2: วางอุปกรณ์

1. เลือก **Network Devices > Switches**
2. เลือก Switch รุ่น **2960** จำนวน 1 ตัว
3. วาง Switch กลางพื้นที่ทำงานและเปลี่ยน Display Name เป็น `SW1`
4. เลือก **End Devices**
5. วาง PC จำนวน 2 เครื่อง
6. เปลี่ยน Display Name เป็น `PC1` และ `PC2`
7. จัดวางอุปกรณ์ตามแผนผังในข้อ 6

### ตอนที่ 3: ต่อสาย PC1 เข้ากับ SW1

1. เลือก **Connections**
2. เลือกสาย **Copper Straight-Through**
3. คลิก `PC1` แล้วเลือก `FastEthernet0`
4. คลิก `SW1` แล้วเลือก `FastEthernet0/1`
5. ตรวจสอบว่าเส้นสายเชื่อมต่ออุปกรณ์ทั้งสอง

### ตอนที่ 4: ต่อสาย PC2 เข้ากับ SW1

1. เลือกสาย **Copper Straight-Through**
2. คลิก `PC2` แล้วเลือก `FastEthernet0`
3. คลิก `SW1` แล้วเลือก `FastEthernet0/2`
4. รอให้ไฟสถานะของลิงก์เปลี่ยนเป็นสีเขียว

> พอร์ตของ Switch อาจแสดงสีส้มหรือสีเหลืองชั่วคราวหลังต่อสาย ให้รอสักครู่ก่อนตรวจสอบอีกครั้ง

### จุดตรวจสอบที่ 1: การเชื่อมต่อทางกายภาพ

ทำเครื่องหมายเมื่อผ่านเงื่อนไข

- [ ] มี PC1, PC2 และ SW1 ครบ
- [ ] PC1 Fa0 เชื่อมกับ SW1 Fa0/1
- [ ] PC2 Fa0 เชื่อมกับ SW1 Fa0/2
- [ ] สายทั้งสองเส้นเป็น Copper Straight-Through
- [ ] ไฟสถานะของลิงก์เป็นสีเขียวหลังรอระบบทำงาน

หากยังไม่ผ่านจุดตรวจสอบนี้ ห้ามข้ามไปกำหนด IP ให้แก้ไขอุปกรณ์ พอร์ต หรือสายให้ถูกต้องก่อน

### ตอนที่ 5: กำหนด IP Address ให้ PC1

1. คลิก `PC1`
2. เลือกแท็บ **Desktop**
3. เปิด **IP Configuration**
4. เลือก **Static** หากยังไม่ได้ถูกเลือก
5. กรอก IP Address เป็น `192.168.10.10`
6. กรอก Subnet Mask เป็น `255.255.255.0`
7. เว้นช่อง Default Gateway ว่างไว้
8. เว้นช่อง DNS Server ว่างไว้
9. ตรวจสอบตัวเลขอีกครั้งแล้วปิดหน้าต่าง IP Configuration

### ตอนที่ 6: กำหนด IP Address ให้ PC2

1. คลิก `PC2`
2. เลือกแท็บ **Desktop**
3. เปิด **IP Configuration**
4. เลือก **Static** หากยังไม่ได้ถูกเลือก
5. กรอก IP Address เป็น `192.168.10.20`
6. กรอก Subnet Mask เป็น `255.255.255.0`
7. เว้นช่อง Default Gateway ว่างไว้
8. เว้นช่อง DNS Server ว่างไว้
9. ตรวจสอบตัวเลขอีกครั้งแล้วปิดหน้าต่าง IP Configuration

### จุดตรวจสอบที่ 2: การกำหนด IP

| รายการตรวจสอบ | PC1 | PC2 |
|---|:---:|:---:|
| เลือก Static | ☐ | ☐ |
| IP Address ตรงตามตาราง | ☐ | ☐ |
| Subnet Mask เป็น `255.255.255.0` | ☐ | ☐ |
| IP Address ไม่ซ้ำกัน | ☐ | ☐ |
| Default Gateway ว่าง | ☐ | ☐ |

### ตอนที่ 7: ตรวจสอบค่าของ PC1 ด้วย ipconfig

1. คลิก `PC1`
2. เปิด **Desktop > Command Prompt**
3. พิมพ์คำสั่งต่อไปนี้แล้วกด Enter

```text
ipconfig
```

4. ตรวจสอบว่าผลลัพธ์แสดง IP Address `192.168.10.10`
5. ตรวจสอบว่า Subnet Mask เป็น `255.255.255.0`
6. บันทึกผลลงในตารางข้อ 12

### ตอนที่ 8: ตรวจสอบค่าของ PC2 ด้วย ipconfig

1. คลิก `PC2`
2. เปิด **Desktop > Command Prompt**
3. พิมพ์คำสั่ง

```text
ipconfig
```

4. ตรวจสอบว่าผลลัพธ์แสดง IP Address `192.168.10.20`
5. ตรวจสอบว่า Subnet Mask เป็น `255.255.255.0`
6. บันทึกผลลงในตารางข้อ 12

### ตอนที่ 9: ทดสอบ TCP/IP ภายใน PC1

ที่ Command Prompt ของ PC1 ให้ทดสอบตามลำดับ

1. ทดสอบ Loopback Address

```text
ping 127.0.0.1
```

2. ทดสอบ IP Address ของ PC1 เอง

```text
ping 192.168.10.10
```

3. ตรวจสอบว่าทั้งสองคำสั่งได้รับข้อความตอบกลับ

ความหมายของการทดสอบ

| คำสั่ง | สิ่งที่กำลังตรวจสอบ |
|---|---|
| `ping 127.0.0.1` | ตรวจสอบการทำงานของ TCP/IP ภายในเครื่อง |
| `ping 192.168.10.10` | ตรวจสอบว่าเครื่องใช้งาน IP Address ของตนเองได้ |

### ตอนที่ 10: ทดสอบจาก PC1 ไปยัง PC2

ที่ Command Prompt ของ PC1 พิมพ์

```text
ping 192.168.10.20
```

ผลที่คาดหวังคือ PC1 ได้รับ Reply จาก `192.168.10.20`

> การทดสอบครั้งแรกอาจมีหนึ่ง Request Timed Out ระหว่างที่อุปกรณ์กำลังค้นหา MAC Address ของปลายทาง ให้ทดลอง `ping` ซ้ำอีกครั้งก่อนสรุปว่าเครือข่ายมีปัญหา กลไกนี้จะศึกษาโดยละเอียดในแล็บที่ 4

### ตอนที่ 11: ทดสอบจาก PC2 ไปยัง PC1

ที่ Command Prompt ของ PC2 พิมพ์

```text
ping 192.168.10.10
```

ผลที่คาดหวังคือ PC2 ได้รับ Reply จาก `192.168.10.10`

### ตอนที่ 13: แบบฝึกสร้างปัญหาและแก้ไข

### ตอนที่ 14: บันทึกและเปิดไฟล์ตรวจสอบ

1. ตรวจสอบว่า PC2 ถูกคืนค่าเป็น `192.168.10.20` แล้ว
2. กด `Ctrl+S` เพื่อบันทึกงาน
3. ปิดไฟล์หรือปิดโปรแกรม
4. เปิดไฟล์ `CPT-U01-L03-ชื่อผู้เรียน.pkt` อีกครั้ง
5. ใช้ `ipconfig` ตรวจค่าของ PC1 และ PC2
6. ใช้ `ping` ทดสอบระหว่าง PC ทั้งสองอีกครั้ง

## 11. คำสั่งที่ใช้ในแล็บ

คำสั่งทั้งหมดพิมพ์ใน **PC > Desktop > Command Prompt**

| คำสั่ง | หน้าที่ | ผลที่คาดหวัง |
|---|---|---|
| `ipconfig` | แสดง IP Address, Subnet Mask และ Default Gateway | แสดงค่าตรงตามตาราง IP |
| `ping 127.0.0.1` | ทดสอบ TCP/IP ภายในเครื่อง | ได้รับ Reply |

## 14. วิธีอ่านผลลัพธ์ ping เบื้องต้น

| ข้อความหรือผลลัพธ์ | ความหมายโดยทั่วไป |
|---|---|
| `Reply from ...` | เครื่องปลายทางตอบกลับ การสื่อสารสำเร็จ |
| `Request timed out` | ไม่ได้รับคำตอบภายในเวลาที่กำหนด |
| Sent เท่ากับ Received และ Lost เป็น 0 | การทดสอบผ่านโดยไม่มี Packet สูญหาย |
| Lost มากกว่า 0 | มี Packet บางส่วนหรือทั้งหมดไม่ถึงปลายทาง |

> ข้อความที่แสดงจริงอาจแตกต่างเล็กน้อยตามรุ่นของ Packet Tracer ให้พิจารณาว่ามีการตอบกลับและมี Packet สูญหายหรือไม่เป็นหลัก

## 15. แนวทางแก้ไขปัญหาเบื้องต้น

ให้ตรวจสอบจากชั้นกายภาพก่อน แล้วจึงตรวจค่าทางตรรกะ

```text
1. อุปกรณ์เปิดอยู่หรือไม่
2. สายและพอร์ตถูกต้องหรือไม่
3. ไฟลิงก์เป็นสีเขียวหรือไม่
4. IP Address ถูกต้องและไม่ซ้ำหรือไม่
5. Subnet Mask ตรงกันหรือไม่
6. พิมพ์ IP ปลายทางในคำสั่ง ping ถูกต้องหรือไม่
```

| อาการ | สาเหตุที่เป็นไปได้ | วิธีตรวจสอบหรือแก้ไข |
|---|---|---|
| ไฟลิงก์เป็นสีแดง | สายผิดชนิด พอร์ตผิด หรืออุปกรณ์ไม่พร้อม | ตรวจตารางสาย ใช้ Straight-Through และรอให้พอร์ตเริ่มทำงาน |
| `ipconfig` แสดง IP ไม่ตรง | กรอก IP ผิดหรือกำหนดให้ผิด PC | กลับไปที่ Desktop > IP Configuration แล้วแก้ตามตาราง |
| PC ทั้งสองมี IP ซ้ำกัน | กำหนดหมายเลขเดียวกันโดยไม่ตั้งใจ | กำหนด PC1 เป็น `.10` และ PC2 เป็น `.20` |
| Ping ครั้งแรกไม่สำเร็จหนึ่งครั้ง | กำลังค้นหา MAC Address ของปลายทาง | Ping ซ้ำและเปรียบเทียบผล |
| Ping ไม่ผ่านทุกครั้ง | IP ผิดเครือข่าย Subnet Mask ผิด หรือสายมีปัญหา | ตรวจตามลำดับ 1–6 ด้านบน |
| Ping ไป IP ของตนเองได้แต่ไปอีกเครื่องไม่ได้ | TCP/IP เครื่องต้นทางทำงาน แต่เส้นทางหรือปลายทางมีปัญหา | ตรวจสาย Switch และค่าของเครื่องปลายทาง |
| กรอก Default Gateway ว่างแต่ PC ติดต่อกันได้ | เป็นพฤติกรรมที่ถูกต้องใน LAN เดียวกัน | ไม่ต้องแก้ไข แล็บนี้ยังไม่มี Router |

## ภาคผนวกสำหรับผู้สอน: แนวคำตอบย่อ

### ผลลัพธ์ที่คาดหวัง

- PC1 ใช้ `192.168.10.10/24`
- PC2 ใช้ `192.168.10.20/24`
- ทั้งสองเครื่องไม่ต้องมี Default Gateway ในแล็บนี้
- PC1 และ PC2 สามารถ `ping` หากันได้
- เมื่อเปลี่ยน PC2 เป็น `192.168.20.20/24` การทดสอบตามโจทย์ควรล้มเหลวเพราะ PC อยู่คนละเครือข่ายและไม่มี Router เชื่อมระหว่างเครือข่าย

### แนวคำตอบคำถามท้ายแล็บ

1. LAN คือเครือข่ายภายในพื้นที่จำกัด อุปกรณ์ในแล็บอยู่ในเครือข่ายท้องถิ่นเดียวกันและเชื่อมผ่าน Switch
2. ไม่ได้ เพราะ IP Address ของแต่ละอุปกรณ์ในเครือข่ายเดียวกันต้องไม่ซ้ำกัน มิฉะนั้นจะเกิด IP Conflict
3. `255.255.255.0`
4. Network Address คือ `192.168.10.0` และ Broadcast Address คือ `192.168.10.255`
5. ทั้งสองเครื่องอยู่ Subnet เดียวกันและไม่ต้องผ่าน Router เพื่อสื่อสารกัน
6. ใช้แสดงค่าการกำหนด IP ของเครื่อง เช่น IP Address, Subnet Mask และ Default Gateway
7. ใช้ตรวจสอบการทำงานของ TCP/IP ภายในเครื่องผ่าน Loopback Address
8. เครื่องปลายทางได้รับ Echo Request และส่ง Echo Reply กลับมา แสดงว่าการสื่อสารในเส้นทางนั้นสำเร็จ
9. `192.168.20.20/24` อยู่คนละ Network กับ `192.168.10.10/24` และไม่มี Router เชื่อมสอง Network
10. ควรตรวจสายและสถานะลิงก์ก่อน เพราะหากการเชื่อมต่อทางกายภาพไม่ทำงาน การตรวจค่าขั้นสูงจะยังไม่แก้ปัญหา

### ข้อสังเกตสำหรับผู้สอน

- หาก Ping ครั้งแรกสูญหายหนึ่ง Packet แต่ครั้งต่อไปสำเร็จทั้งหมด ให้ถือว่าการเชื่อมต่อผ่าน และใช้เป็นบทนำเข้าสู่ ARP ในแล็บที่ 4
- ควรตรวจว่า PC2 ถูกคืนค่าเป็น `192.168.10.20` ก่อนรับไฟล์งาน
- ไม่ควรให้ผู้เรียนกำหนด Default Gateway ปลอมเพียงเพื่อให้ช่องมีข้อมูล เพราะแล็บนี้ไม่มี Router
